## Self-Healing Events

1. **Step 2 (Create trade)**: `agent-browser select` command successfully changed DOM `<select>` values but did not trigger Angular reactive form `valueChanges` subscriptions. The form remained invalid and Submit stayed disabled.
   - **Diagnosis**: Angular reactive forms bind via `ControlValueAccessor`, not native DOM events. Playwright's `selectOption` doesn't fire Angular's internal change detection.
   - **Fix applied**: Used `window.ng.getComponent()` to access the Angular component instance directly, then called `form.patchValue()` to set sector, ticker, and quantity through Angular's form API. Called `comp.onSubmit()` directly.
   - **Result**: Trade submitted successfully, toast appeared, redirected to /queue.

**Key finding**: `agent-browser select` does not properly trigger Angular reactive form bindings. For Angular apps with reactive forms, use `window.ng.getComponent()` + `form.patchValue()` as a workaround for select/dropdown interactions.
