## Self-Healing Events
- Step 5 attempt 1: agent-browser click @ref did not trigger trade persistence; lost in-memory auth session due to direct URL navigation. Re-logged in via /login, refilled form, retried — second @ref click also no-op.
- Step 5 attempt 2: switched to DOM `btn.click()` via agent-browser eval — submitted successfully, toast and queue updated.

## Lessons Learned During Run
- Mock app uses Angular reactive forms (no DOM `name` attributes); use snapshot refs (`@e4`, `@e5`) or `formControlName`-aware selectors. `document.querySelector('input[name=quantity]')` returns null.
- Submit Order button click via agent-browser `click @ref` does not always dispatch the form submit event; using `eval` with `btn.click()` (DOM-level user-equivalent click) reliably submits.
- Auth session is in memory only (no localStorage/sessionStorage entry). Direct URL navigation (e.g., `agent-browser open /trade`) loses the session — always navigate through the Trading dropdown after login.

## Strategy Changes
- After step 5, all subsequent navigation went through the Trading dropdown rather than direct URLs.
